# Implementation notes

Task created from the observation that `scry` should apply the Testing Without Mocks pattern language to its REPL verification strategy. Recent REPL runs showed that broad discovery in a long-lived REPL can mix intentional failure fixtures, optional Kaocha classpath behavior, generated temporary-project namespaces, and stale process-global state. The design captures a narrow, sociable, state-based test strategy before implementation planning.

2026-06-03 architecture review: no new actionable architectural-fit feedback. The design aligns with scry's REPL-first structured-result purpose, preserves core/Kaocha dependency boundaries, keeps public result shapes stable, treats CI/CLI verification as separate final/process checks, and uses thin nullable boundaries only for unavoidable external effects. `META.md` and `doc/architecture.md` were absent, so review used `AGENTS.md` plus the task design.

2026-06-03 ambiguity review: found actionable ambiguity feedback. The design still leaves planning-critical choices open around how canonical REPL slices are represented, exact alias/classpath REPL contexts and snippets, whether CI should keep broad discovery or switch to focused slices, the temp namespace cleanup strategy for Kaocha/project fixtures, and the inventory/acceptance criteria for remaining Nullable infrastructure boundaries versus mock-like injections.

2026-06-03 ambiguity follow-up: resolved all review-added design questions in `design.md`. The canonical REPL mechanism is documentation-only explicit `scry/run` snippets for now; exact `:test`, `:test:kaocha`, `:test:build`, and `:test:release-test` alias contexts/forms are pinned; CI is directed toward explicit focused slices with any broad discovery run only a smoke check; temporary Kaocha/project namespaces should use unique generated names plus `remove-ns`; and accepted Nullable boundaries are limited to CLI output/cwd/System-exit seams, real-runner application seam use, Git/process boundaries, deploy, and release command execution with state/output assertions.

2026-06-03 inconsistency review: found actionable inconsistency feedback. The design also requires AGENTS.md to explain focused slices/broad discovery, but current AGENTS.md still presents broad `(scry/run)` as the way to run the current REPL test suite; implementation needs an explicit replace-or-qualify direction.

2026-06-03 inconsistency follow-up: reconciled the CI and AGENTS guidance in `design.md`.
