# Implementation notes

Task created from the maintainer question about making capture state nullable in the sense of replaceable: whenever a test is run within a test, the nested runner should install its own capture state or disable the enclosing capture, so inner fixture events do not leak into the outer result. The task also records the related desire to rebind `*out*` and `*err*` around each core test var for stronger per-test output isolation.
